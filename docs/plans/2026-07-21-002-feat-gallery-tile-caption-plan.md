# Gallery tile module and section caption

## Goal Capsule

- **Objective:** Add a caption above each diagram preview in the static image gallery that names the originating deep-dive module and section.
- **Authority hierarchy:** The generated `gallery.html` and `scripts/generate-gallery.py` are the source of truth for gallery rendering; changes must survive regeneration.
- **Stop conditions:** Stop when every gallery card shows a module/section caption above its SVG, the generator produces identical output on re-run, and the visual result is verified in a browser.
- **Execution profile:** Lightweight frontend change; one implementation unit plus verification.
- **Tail ownership:** `ce-work` owns implementation and verification.

## Product Contract

### Summary

The diagram gallery renders one tile per figure extracted from deep-dive companions. Each tile currently shows the SVG, a title, the section name, and a "why" note below the image. Stakeholders want the tile to surface provenance at a glance by placing a compact caption directly above the image that states the originating module (deep-dive file) and section.

### Requirements

- R1. Every gallery card must display a caption above the diagram preview.
- R2. The caption must show the originating deep-dive module name and the section title from the registry entry.
- R3. The caption must be generated by `scripts/generate-gallery.py` so `gallery.html` stays reproducible.
- R4. The caption styling must match the existing gallery typography (monospace labels, soft color, compact spacing).
- R5. Existing card behaviour (filtering, lightbox, keyboard navigation, link to deep dive) must remain unchanged.

## Planning Contract

### Key Technical Decisions

- KTD1. Caption lives inside the card above `.gallery-preview` rather than inside the SVG. This keeps the SVG untouched, preserves the lightbox clone logic, and lets CSS style the caption as plain HTML.
- KTD2. The "module" label is derived from the registry entry's `deepDive` stem (e.g. `B1-bootstrap-paradox-deepdive`) without inventing a new friendly-name field. It matches the file/module identity used in the deep-dive link.
- KTD3. Caption text is added to the card's searchable `data-*` attributes so filtering by module or section continues to work after the change.

### Assumptions

- The gallery is rebuilt by running `scripts/generate-gallery.py`; hand-edits to `gallery.html` would be overwritten.
- A human-readable module name is not required; the stem is sufficient for identifying the source deep dive.

## Implementation Units

### U1. Generate module-and-section caption in gallery tiles

- **Goal:** Update the gallery generator and its CSS so each card shows a compact provenance caption above the diagram preview.
- **Requirements:** R1, R2, R3, R4, R5.
- **Dependencies:** None.
- **Files:**
  - `scripts/generate-gallery.py`
  - `gallery.html`
- **Approach:**
  - Add a `.gallery-caption` CSS rule to `GALLERY_CSS` in `scripts/generate-gallery.py`: small monospace uppercase label, soft color, padding above the preview, and a subtle bottom border or spacing that separates it from the SVG.
  - Modify `render_card` to emit a `<div class="gallery-caption">` before `.gallery-preview`. Build the caption text conditionally: `{module} · {section}` when `section` is non-empty, otherwise just `{module}`, so no stray separator appears. Pass the composed caption through the existing `escape_html()` helper before interpolating it into the markup and the `data-*` attributes.
  - Add a `data-module` attribute to each card containing the lower-cased module stem, and update the filter expression in the generated JavaScript to include `card.dataset.module`. Update the filter input's `placeholder` and `aria-label` to mention module as a searchable dimension.
  - Regenerate `gallery.html` by running the generator.
- **Patterns to follow:** Existing `.gallery-section` styling uses `--mono` and `--soft`; mirror that for the caption.
- **Test scenarios:**
  - Happy path: open `gallery.html` in a browser and confirm every card has a caption above the SVG containing the module stem and section text.
  - Edge case: entries with an empty `section` still render a caption that shows only the module stem and no stray separators.
  - Integration scenario: type a module stem or section name into the filter input and verify the corresponding cards remain visible.
- **Verification:** Browser visual check; re-run `python3 scripts/generate-gallery.py` and confirm `gallery.html` is byte-identical to the committed version (no drift between generated and hand-edited output).

## Verification Contract

| Gate | Command / Check | Expected result |
|---|---|---|
| Generator reproducibility | `python3 scripts/generate-gallery.py` | After the regenerated `gallery.html` is committed, re-running the generator produces no further diff (exit 0, no diff) |

| Behaviour preserved | Use filter input and lightbox | Filtering and lightbox still work for all cards |

## Definition of Done

- `scripts/generate-gallery.py` emits the new caption and can regenerate `gallery.html` without changes.
- `gallery.html` shows a module-and-section caption on every card.
- Filter, lightbox, keyboard navigation, and deep-dive links still work.
- No absolute paths, no new dependencies, and no dead experimental code remain.
